# 2026-06-02 Release

Release version: `2026.6.2`

This release tightens the KiCad SVG and design-review surfaces after the first
public baseline. It is intended to support initial downstream review tooling
with better PCB/SVG parity, richer SVG metadata, and more ergonomic schematic
hierarchy traversal.

## Added

- PCB SVG render profiles for oracle-style comparison and enriched review
  output.
- Enriched PCB SVG metadata for board stackup, project text variables,
  components, pads, vias, tracks, zones, drills, and net linkage.
- Enriched schematic SVG metadata with embedded KiCad-native design JSON and
  view-local SVG-to-net indexes.
- Public `KiCadSchematicInstance` records plus `KiCadDesign` helpers for
  enumerating sheet instances, finding where a schematic file is used,
  navigating parent/child sheet relationships, and rendering one concrete
  schematic instance to IR.
- `uv.lock` for reproducible uv development and CI environments.

## Fixed

- PCB SVG output now more closely matches KiCad CLI behavior for custom pads,
  NPTH mask apertures, filled polygons, thick segments, stroke widths, track
  arc direction, render-cache fill rules, dimensions, and review-layer naming.
- Design and schematic hierarchy contract revisions are normalized to `a0`.
- Schematic SVG net linkage now includes view-local indexes so repeated
  hierarchical sheets can resolve graphical groups to the correct net context.

## Tests And Signoff

- Added strict PCB SVG structural-oracle checks and broadened promoted
  synthetic and real-world SVG parity coverage.
- Added schematic SVG linkage tests for enriched metadata and view-local
  indexes.
- Added synthetic and taillight real-world coverage for repeated schematic
  sheet instance traversal.
- Updated README examples for design loading, netlist extraction, SVG
  rendering, object queries, and focused model mutation.

## Known Follow-Up Work

- Continue using downstream tooling integration to decide which provisional
  package-root exports should graduate into the promoted public API contract.
- Continue reviewing public corpus outputs visually as new real-world fixtures
  are promoted.
